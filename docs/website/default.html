01. Nervous System

JSON Router & IO Layer

The high-performance core (RT.gateway) handles message routing through three distinct layers, ensuring that business logic never touches networking complexity.

Layered Identity

  • L1 (Socket): UUID-based physical delivery.
  • L2 (Process): Descriptive naming (TYPE.name@island).
  • L3 (Interlocutor): ILK (Interlocutor Key) for human/agent identity.

IO Nodes

Adapters (WhatsApp, Email, Slack) that normalize external traffic into the internal protocol.

They resolve external IDs into ICH (Interlocutor Channels) and derive the CTX context automatically.

02. Cognitive Engine

Memory & SY.cognition

fluxbee doesn't just route; it remembers. Every interaction is consolidated into episodic memory packages available for AI nodes in real-time.

Episodic Memory

Experience is grouped into Events (episodes) with clear boundaries.

  • LanceDB: Embedded vector cache for micro-latency retrieval.
  • ctx_window: The last 20 turns injected into every message header.

SY.cognition

The single-writer process that manages jsr-memory. It extracts semantic tags and adjusts activation_strength based on successful outcomes.

03. Cluster Management

Motherbee & Workers

A strict separation between the persistent Brain (Motherbee) and the stateless Muscle (Workers) allows for infinite horizontal scaling.

Motherbee (Primary)

The source of truth. Runs SY.orchestrator, SY.storage, and the PostgreSQL authority.

role: motherbee
database: postgres://...
wan.listen: "0.0.0.0:9000"

Stateless Workers

Workers are rebuildable. They sync identity/config/opa from the Motherbee via the WAN bridge.

If a worker dies, the Orchestrator performs a cold start, rebuilding its cache from the Primary Hub.

04. Reliability

Storage & SY.admin

NATS Persistence

The router features an embedded NATS JetStream. It uses a "fire & forget" pattern: the router publishes to NATS in ~1ms, while SY.storage persists to DB asynchronously.

SY.admin & OPA

The única API global. Distributes CONFIG_CHANGED messages via broadcast.

Policies are compiled from Rego to WASM and shared via the jsr-opa SHM region.
